## Reporting Security Vulnerabilities

If you discover a security vulnerability in this project:

### DO NOT:
- ❌ Open a public issue
- ❌ Submit a pull request
- ❌ Disclose the vulnerability publicly
- ❌ Share details on social media or forums

### DO:
1. ✅ Contact the repository owner **privately**:
   - GitHub: @chavanarya36
   - Use GitHub's private vulnerability reporting feature (if available)
   - Or DM via GitHub Discussions

2. ✅ Include in your report:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact assessment
   - Suggested fix (optional)

3. ✅ Allow reasonable time for response and patching (typically 7-14 days)

## Responsible Disclosure

We appreciate responsible disclosure practices:
- Report vulnerabilities privately first
- Allow time for patches before public disclosure
- Work with us to understand the issue

## What We Will Do

Upon receiving a valid security report:
1. Acknowledge receipt within 48 hours
2. Investigate and confirm the issue
3. Develop and test a fix
4. Release a patch
5. Credit you for the discovery (if desired and appropriate)

## Scope

This security policy covers:
- The CloudGuard AI web application (app.py)
- ML model security and integrity
- Data handling and privacy
- Deployment configurations
