# CloudGuardAI: Graph-Augmented Static Analysis for Infrastructure-as-Code Security

CloudGuardAI is a static analysis platform for detecting security misconfigurations in Terraform infrastructure-as-code (IaC) files. It addresses the problem that conventional IaC scanners operate on individual resources in isolation, producing flat lists of findings without contextual relationships—making it difficult for engineers to assess which misconfigurations pose genuine risk and which are low-priority noise.

The system combines three complementary analysis layers. First, a deterministic rules engine applies 34 pattern-matching rules across categories including network exposure, storage encryption, access control, secrets management, and database hardening. Second, a graph neural network (GNN) constructs a resource dependency graph from each Terraform file and performs multi-hop traversal to identify attack paths—chains of connected misconfigurations where an exposed entry point (e.g., an unrestricted security group) leads to a high-value target (e.g., an unencrypted database with hardcoded credentials). Third, a unified findings layer scores and ranks all detections, enriching each finding with a reasoning summary, an attack-path membership flag, and contextual metadata describing the finding's role in any identified chain.

The key differentiator is the graph-based attack path analysis. Rather than reporting each misconfiguration independently, the system surfaces multi-resource vulnerability chains and prioritizes findings that participate in exploitable paths. This produces a recommended finding order that reflects actual risk topology, not just individual severity labels.

Outputs include per-finding severity, category, remediation steps, and—for attack-path findings—a structured context object identifying upstream and downstream resources, hop count, and the finding's role (entry point, intermediate, or target). A scan-level recommended ordering guides triage.

The system is validated against a curated 50-file evaluation dataset (25 vulnerable, 25 clean) with ground-truth labels spanning 11 vulnerability categories. Evaluation uses optimal bipartite matching between predicted and labeled findings. Reproducible metrics on the frozen version-1 dataset are: precision 0.904, recall 0.940, F1 0.922. Baseline comparisons against Checkov and tfsec are included, demonstrating that the system detects 18 vulnerability instances not found by either tool, primarily through its graph-based multi-resource analysis.
