# Contributing to CloudGuard AI

## ⚠️ IMPORTANT: This is a Proprietary Project

**CloudGuard AI is proprietary software.** This project is **NOT open for public contributions**.

## Contribution Policy

### ❌ We DO NOT Accept:
- Unsolicited pull requests
- Code contributions from external contributors
- Feature requests via PRs
- Bug fix submissions without prior authorization

### ✅ Authorized Contributions Only:
Contributions are accepted **only if**:
1. You have been **explicitly invited** by the repository owner (chavanarya36)
2. You have signed a **Contributor License Agreement (CLA)**
3. Your contribution was **specifically requested**

## Why This Policy?

This project represents original research and development work by chavanarya36. To maintain:
- **Intellectual property rights**
- **Academic integrity**
- **Professional credit**
- **Legal protection**

...we cannot accept external contributions without formal agreements.

## Reporting Issues

If you discover security vulnerabilities or critical bugs:

1. **DO NOT** open a public issue or pull request
2. **Contact** the repository owner privately:
   - Via GitHub: @chavanarya36
   - Email: [Contact via GitHub profile]
3. **Wait** for authorization before submitting any code

## Questions or Feature Suggestions?

- Open a **discussion** (not an issue) to ask questions
- Feature suggestions may be noted but are not guaranteed to be implemented
- You may **fork for personal study only** (not for redistribution or public hosting)

## Contributor License Agreement (CLA)

If you are invited to contribute, you will be required to sign a CLA stating:
- You assign copyright of your contributions to the project owner
- You grant full rights to use, modify, and distribute your contributions
- Your contributions will become part of the proprietary codebase
- You will not claim credit or ownership of the final work

- The repository owner retains all rights
